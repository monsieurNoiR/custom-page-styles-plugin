# セキュリティチェックリスト

このドキュメントは、Studio Noir Custom Page Stylesプラグインに実装されているセキュリティ対策の詳細なチェックリストです。

## 1. 認証と認可 (Authentication & Authorization)

### ✅ 実装済み

- **権限チェック**
  - `current_user_can('manage_options')` - 設定ページアクセス
  - `current_user_can('edit_post', $post_id)` - 投稿編集権限
  - `current_user_can('edit_post', $post_id)` - AJAXファイルアップロード (v1.1.0)
  - `current_user_can('edit_post', $post_id)` - AJAXファイル削除 (v1.1.0)

- **Nonce検証**
  - メタボックス保存時のNonce検証
  - WordPress設定API使用（自動的にNonce処理）
  - AJAXファイルアップロード時のNonce検証 (v1.1.0)
  - AJAXファイル削除時のNonce検証 (v1.1.0)

## 2. 入力検証とサニタイゼーション (Input Validation & Sanitization)

### ✅ 実装済み

- **POSTデータ処理**
  - `wp_unslash()` - スラッシュ除去
  - `absint()` - 整数検証
  - `sanitize_key()` - キー値のサニタイゼーション
  - `sanitize_file_name()` - ファイル名のサニタイゼーション (v1.1.0)

- **CSS入力のサニタイゼーション**
  - `wp_strip_all_tags()` - HTMLタグ除去
  - 危険なCSSパターンの削除:
    - `@import` - 外部ファイル読み込み防止
    - `javascript:` - JavaScriptプロトコル防止
    - `expression()` - IE特有の脆弱性防止
    - `behavior:`, `-moz-binding` - バインディング攻撃防止
  - 括弧バランスチェック

- **投稿タイプ配列のサニタイゼーション**
  - 登録済み投稿タイプとの照合

- **アップロードファイル検証 (v1.1.0)**
  - ファイルタイプ検証（CSS/JSのみ許可）
  - ファイルサイズ制限（5MB以内）
  - ファイル名のサニタイゼーション
  - 拡張子のホワイトリスト検証

- **アップロードファイル設定のサニタイゼーション (v1.1.0)**
  - `filename` のサニタイゼーション (`sanitize_file_name()`)
  - `type` のホワイトリスト検証（css/js）
  - `load_in` のホワイトリスト検証（header/footer）

## 3. 出力エスケープ (Output Escaping)

### ✅ 実装済み

- **HTML出力**
  - `esc_html()` - HTMLコンテキスト
  - `esc_attr()` - 属性値
  - `esc_textarea()` - テキストエリア
  - `esc_html__()` / `esc_html_e()` - 翻訳文字列
  - `esc_js()` - JavaScript内文字列 (v1.1.0)

- **JavaScriptでのXSS対策 (v1.1.0)**
  - `.text()` - テキスト挿入（自動エスケープ）
  - `.val()` - 値設定（自動エスケープ）
  - 直接的なHTML挿入を回避

## 4. SQLインジェクション対策 (SQL Injection Prevention)

### ✅ 実装済み

- **プリペアドステートメント**
  - `$wpdb->prepare()` 使用
  - プレースホルダー使用 (%s, %d)
  - すべてのユーザー入力をバインド

## 5. ファイルシステムセキュリティ (File System Security)

### ✅ 実装済み

- **安全なファイル操作**
  - WordPress Filesystem API使用
  - `WP_Filesystem()` による抽象化
  - 直接的な `file_put_contents()` は不使用

- **パストラバーサル攻撃対策**
  - `realpath()` によるパス検証
  - `absint()` による投稿ID検証
  - ファイル名の動的生成制限
  - アップロードファイル名のサニタイゼーション (v1.1.0)
  - enqueue時のファイル名サニタイゼーション (v1.1.0)

- **ディレクトリ保護**
  - `index.php` 配置 - ディレクトリリスティング防止
  - `.htaccess` 配置 - PHP実行防止
  - 適切なファイルパーミッション (`FS_CHMOD_FILE`)

- **投稿別ディレクトリ分離 (v1.1.0)**
  - `/sn-cps-styles/{post_id}/` でファイル隔離
  - ファイル名衝突の防止
  - 投稿削除時の一括クリーンアップ対応

- **ファイルアップロードセキュリティ (v1.1.0)**
  - `move_uploaded_file()` による安全なアップロード
  - アップロード先ディレクトリの検証
  - ファイルサイズ制限（5MB）
  - MIMEタイプ検証（finfo_file） (v1.1.1)
  - **ファイル名衝突時の自動リネーム** (v1.1.1)

## 6. XSS (Cross-Site Scripting) 対策

### ✅ 実装済み

- **すべての出力をエスケープ**
  - 管理画面表示
  - メタボックスレンダリング
  - 設定ページ表示

- **HTML/JavaScriptの除去**
  - CSS内のJavaScriptプロトコル除去
  - `wp_strip_all_tags()` によるタグ除去

- **JavaScriptでのDOM操作 (v1.1.0)**
  - `.text()` / `.val()` 使用による自動エスケープ
  - `.html()` の直接使用を回避
  - ユーザー入力を直接DOM挿入しない

## 7. CSRF (Cross-Site Request Forgery) 対策

## 推奨される追加対策（オプション）

プラグインは既に本番環境で使用可能なセキュリティレベルですが、さらに強化したい場合:

1. **Content Security Policy (CSP)**
   - 管理画面でのCSPヘッダー追加

2. **レート制限**
   - API呼び出しや保存操作の頻度制限
   - ファイルアップロードの頻度制限

3. **監査ログ**
   - CSS変更履歴の記録
   - ファイルアップロード履歴の記録

4. **自動スキャン**
   - CSS内容の定期的なセキュリティスキャン
   - アップロードファイルのマルウェアスキャン

5. **2要素認証統合**
   - 重要操作での追加認証

## セキュリティ報告

セキュリティ上の問題を発見した場合は、以下の手順で報告してください:

1. 公開せず、非公開で報告
2. 詳細な再現手順を提供
3. 影響範囲の評価
4. 可能であれば修正案を提案

## 更新履歴

### v1.1.1 (2025-02-08) - セキュリティ強化アップデート
- **強化: パストラバーサル対策の厳格化**
  - `ajax_remove_file()` に `realpath()` による厳格なパス検証を追加
  - 削除対象ファイルが意図したディレクトリ内に存在することを確認
  - ディレクトリトラバーサル攻撃による意図しないファイル削除を完全にブロック

- **新機能: MIMEタイプ検証**
  - `ajax_upload_file()` に `finfo_file()` による実ファイル内容のMIMEタイプ検証を追加
  - 拡張子偽装されたファイル（例: .js.php）のアップロードを防止
  - 許可MIMEタイプ: text/css, text/plain, text/javascript, application/javascript, application/x-javascript

- **新機能: ファイル名衝突の自動解決**
  - 同名ファイルアップロード時に自動リネーム機能を実装
  - ファイル名に連番を付与（例: style.css → style-1.css → style-2.css）
  - 最大100個まで自動リネーム対応（100個超過時はエラー）

### v1.1.0 (2025-02-07) - ファイルアップロード機能追加
- **新機能: ファイルアップロードセキュリティ**
  - ファイルタイプ検証（CSS/JSのみ許可）
  - ファイルサイズ制限（5MB）
  - ファイル名サニタイゼーション（パストラバーサル対策）
  - 投稿別ディレクトリ分離（`/sn-cps-styles/{post_id}/`）
  - `move_uploaded_file()` による安全なアップロード

- **新機能: AJAX処理のセキュリティ**
  - `ajax_upload_file()` - 完全なNonce/権限/入力検証
  - `ajax_remove_file()` - 完全なNonce/権限/入力検証
  - `wp_send_json_error()` による安全なエラー応答

- **強化: XSS対策**
  - JavaScriptでのDOM操作に `.text()` / `.val()` 使用
  - 直接的なHTML挿入を回避
  - ユーザー入力の自動エスケープ

- **強化: パストラバーサル対策**
  - `enqueue_custom_styles()` でファイル名をサニタイゼーション
  - メタデータから取得したファイル名も検証

- **強化: 入力検証**
  - アップロードファイル設定の完全なサニタイゼーション
  - `type` / `load_in` のホワイトリスト検証
